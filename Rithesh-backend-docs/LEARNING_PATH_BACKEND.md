## Learning Path (Backend-focused)

Use this along with daily tasks.

## Phase 1: Go and HTTP basics (Day00-Day03)

- Go Tour: `https://tour.golang.org/`
- Effective Go: `https://go.dev/doc/effective_go`
- Go by Example: `https://gobyexample.com/`
- `net/http`: `https://pkg.go.dev/net/http`
- `encoding/json`: `https://pkg.go.dev/encoding/json`

## Phase 2: Database and migrations (Day01-Day04)

- Postgres docs: `https://www.postgresql.org/docs/`
- SQL tutorial: `https://www.sqltutorial.org/`
- goose migrations: `https://github.com/pressly/goose`
- pgx docs: `https://github.com/jackc/pgx`

## Phase 3: API design and backend logic (Day05-Day10)

- HTTP methods and status: `https://developer.mozilla.org/en-US/docs/Web/HTTP`
- REST basics: `https://restfulapi.net/`
- chi router: `https://github.com/go-chi/chi`

## Phase 4: Auth and security basics (Day11-Day12)

- OWASP Authentication Cheat Sheet: `https://cheatsheetseries.owasp.org/cheatsheets/Authentication_Cheat_Sheet.html`
- OWASP Password Storage Cheat Sheet: `https://cheatsheetseries.owasp.org/cheatsheets/Password_Storage_Cheat_Sheet.html`
- Cookies: `https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies`

## Phase 5: LLM service integration (Day13-Day15)

- FastAPI tutorial: `https://fastapi.tiangolo.com/tutorial/`
- Pydantic: `https://docs.pydantic.dev/latest/`
- Go HTTP client: `https://pkg.go.dev/net/http`
- Contract testing with `httptest`: `https://pkg.go.dev/net/http/httptest`

## Phase 6: Reliability, tests, observability (Day16-Day20)

- Go context: `https://pkg.go.dev/context`
- Go testing: `https://pkg.go.dev/testing`
- Table-driven tests: `https://go.dev/wiki/TableDrivenTests`
- OpenAPI spec: `https://swagger.io/specification/`
- Swaggo: `https://github.com/swaggo/swag`

## Phase 7: Maintainability and hardening (Day21-Day27)

- Go project layout ideas: `https://github.com/golang-standards/project-layout`
- Postgres indexes: `https://www.postgresql.org/docs/current/indexes.html`
- Explain plans: `https://www.postgresql.org/docs/current/using-explain.html`

## Phase 8: Docs and handover (Day28-Day30)

- Technical writing for developers: `https://www.writethedocs.org/guide/`

## Study guidance

- Spend at least 20-45 min/day on reading links.
- Build first, then read deeper when blocked.
- Keep notes of what you learned and where you got stuck.
