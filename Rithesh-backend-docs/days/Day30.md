# Day 30 - Final backend demo + handover

## Goal (end of day)
- backend is demo-ready
- swagger is accurate
- LLM contracts are clear
- next-step backlog exists

## Learn (15-30 minutes)
- Technical handover best practices

## Tasks
### 1) Demo script
Create `Rithesh-backend-docs/DEMO_SCRIPT.md` with exact command sequence:
- start DB
- run API
- run LLM
- open Swagger
- run full story

You can start from:
- `Rithesh-backend-docs/DEMO_SCRIPT_TEMPLATE.md`

### 2) Handover notes
Create `Rithesh-backend-docs/HANDOVER.md`:
- env vars
- ports
- troubleshooting
- out-of-scope items

You can start from:
- `Rithesh-backend-docs/HANDOVER_TEMPLATE.md`

### 3) V2 backlog
Create `Rithesh-backend-docs/V2_BACKLOG.md`:
- real LLM prompt quality + eval loop
- real crowd provider
- real POI search + caching
- production auth hardening

You can start from:
- `Rithesh-backend-docs/V2_BACKLOG_TEMPLATE.md`

### 4) Final self-review
Use:
- `Rithesh-backend-docs/WEEKLY_CHECKPOINTS.md`
- `Rithesh-backend-docs/MENTOR_CHECKLIST.md`

## Acceptance checks
- demo runs from script with no improvisation
- handover docs are enough for another developer to continue work
