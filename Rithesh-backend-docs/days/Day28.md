# Day 28 - Documentation (backend runbook)

## Goal (end of day)
- another developer can run API + DB + LLM from docs only

## Learn (15-30 minutes)
- Runbook writing principles

## Tasks
### 1) Create workspace runbook
Create `README_BACKEND.md` in workspace root (same level as `Rithesh-backend-docs`) using:
- `Rithesh-backend-docs/README_BACKEND_TEMPLATE.md`

### 2) Fill with real project values
Include:
- exact env vars
- exact ports
- exact run commands
- exact script flow

### 3) Add troubleshooting section
Use failures you actually hit in this project.

### 4) Peer test (important)
Have another person (or future-you after clean terminal restart) follow docs without improvising.

## Acceptance checks
- fresh terminal setup works in under 15 minutes
- no undocumented manual step is needed

## Deliverables
- reliable backend runbook in workspace root.
